Vue2-part2-computed

一、computed 计算属性

1.概念

基于现有的数据，计算出来的新属性。依赖的数据变化，自动重新计算

2.语法

  1. 生命在 computed 配置项中，一个计算属性对应一个函数
  2. 使用起来和普通属性一样使用 {{计算属性名}}

3.注意

  1. computed配置项和 data是同级的
  2. computed中的计算属性虽然是函数的写法，但他依然是个属性
  3. computed中的计算属性不能和 data 中的属性同名
  4. computed中的计算属性和使用 data 中的属性是一样的用法
  5. computed中计算属性内部的this 依然指向的是 Vue 实例

4.案例

二、computed计算属性 vs methods 方法

1.computed 计算属性

作用：封装了一段对于数据的处理，求的一个结果

语法：

  1. 写在 computed 配置项中
  2. 作为属性，直接使用
    • js中使用计算属性：this.计算属性
    • 模版中使用计算属性：{{计算属性}}

2.methods 计算属性

作用：给 Vue 实例提供一个方法，通过调用来处理业务逻辑

语法：

  1. 写在 methods 配置项中
  2. 作为方法调用
    • js 中调用：this.方法名()
    • 模版中调用{{方法名()}} 或者 @事件名="方法名"

3.计算属性的优势

  1. 缓存特效（提升性能）
    • 计算属性会对计算出来的结果缓存，再次使用直接读取缓存
    • 依赖项一变化，会自动重新计算 => 并再次缓存
  2. methods 没有缓存特性
