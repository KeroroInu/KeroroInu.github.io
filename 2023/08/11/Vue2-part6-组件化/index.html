Vue2-part6-组件化

一、组件化开发

组件化：一个页面可以拆分成一个个组件，每个组件有这自己独立的结构、样式、行为

好处：便于维护，利于复用 => 提升开发效率

组件分类：普通组件、根组件

二、根组件 App.vue

1.根组件介绍

整个应用最上层的组件，包裹所有普通小组件

2.组件是由三部分构成

  • template：结构（有且只能有一个根元素）
  • script：js 逻辑
  • style：样式（可支持 less，需要装包）

让组件支持 less

  • style 标签，lang=”less” 开启less 功能
  • 装包：yarn add less less-loader -D或者npm i less less-loader -D

四、普通组件的注册使用-全局注册

1.特点：

全居注册的组件，在项目的任何组件中都能使用

2.步骤：

  1. 创建.vue组件（三个组成部分）
  2. Main.js中进行全局注册

5.语法：

Vue.component('组件名', 组件对象)

五、scoped 解决样式冲突

1.默认情况：

写在组件中的样式会 全局生效 => 因此很容易造成多个组件之间的样式冲突问题

  1. 全局样式：默认组件中的样式会做用到全局，任何一个组件中都会受到此样式的影响
  2. 局部样式：可以给组件加上scoped属性，可以让样式作用于当前组件

2.scoped 原理

  1. 当前组件内标签都被添加data-v-hash值的属性
  2. css 选择器都被添加[data-v-hash值]的属性选择器

最终效果：必须是当前组件的元素，才会有这个自定义属性，才会被这个样式作用到

六、data必须是一个函数

1.data 为什么要写成函数

一个组件的 data 选项必须是一个函数。

目的：保证每个组件实例，维护独立的一份数据对象

每次创建新的组件实例，都会新执行一次 data 函数，得到一个新对象

七、组件通信

1.什么是组件通信？

组件通信，就是组件和组件之间的数据传递

  • 组件的数据是独立的，无法直接访问其他组件的数据
  • 想使用其他组件的数据，就需要组建通信

2.组件之间的关系

  1. 父子关系
  2. 非父子关系

3.通信解决方案

  • 父子关系

    • props和$emit

  • 非父子关系

    • proveide&inject
    • eventbus

  • 通用解决方式

    • vuex

4. 父子通信流程

  1. 父组件通过props将数据传递给子组件
  2. 子组件利用$emit通知父组件修改更新

十一、props&data、单向数据流

1.公共点

都可以给组件提供数据

2.区别

  • data 的数据是自己的 => 随便改
  • Props 的数据是外部的 => 不能直接改，要遵循单向数据流

3.单向数据流

父级props 的数据更新，会向下流动，影响子组件。这个数据流动是单向的

十二、非父子通信-event bus 事件总线

1.作用

非父子组件之间，进行简易消息传递。（复杂场景 => Vuex）

2.步骤

1.创建一个都能访问的事件总线（空 Vue 实例）

2.A组件（接收方），监听 Bus 的$on事件

3.B组件（发送方），触发 Bus 的$emit事件

十三、非父子通信-provide&inject

1.作用

跨层级共享数据

2.语法

1.父组件 provide 提供数据

2.子/孙组件 inject 获取数据

3.注意

  • provide 提供的简单类型的数据不是响应式的，复杂类型数据是响应式
    • 推荐提供复杂类型数据
  • 子/孙组件通过 inject 获取的数据，不能在自身组件内修改
