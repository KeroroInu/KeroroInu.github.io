Vue2-part14-Vuex

一、Vuex 概述

1.Vuex 是什么

Vuex 是一个 Vue 的状态管理工具，状态就是数据

大白话：Vuex 是一个插件，可以帮我们管理 Vue 通用的数据（多组件共享的数据）例如：购物车数据 个人信息数据

2.使用场景

  • 某个状态 在 很多个组件 来使用（个人信息）
  • 多个嘴贱 共同维护 一份数据（购物车）

3.优势

  • 共同维护同一份数据，数据集中化管理
  • 响应式变化
  • 操作简介（vuex 提供了一些辅助函数）

4.注意：

官方原文：

  • 不是所有的场景都适用于 vuex，只有在必要的时候才使用 vuex
  • 使用了 vuex 之后，会附加更多的框架中的概念进来，增加了项目的复杂度（数据的操作更便捷，数据的流动更清晰）

Vuex就像《近视眼镜》，你自然会知道什么时候需要用它

十一、核心概念 - module

由于使用单一状态树，应用的所有状态会集中到一个比较大的对象。当应用变得非常复杂时，store 对象就有可能变得相当臃肿

为此，可以将 Vuex 模块化

1.模块定义 - 准备 state

定义两个模块 user 和 setting

user 中管理用户的信息状态 userInfo modules/user.js

setting 中管理项目应用的 主题色 theme，描述 desc modules/setting.js

在store/index.js文件中的 modules 配置项中，注册这两个模块

使用模块中的数据，可以直接通过模块名访问$store.state.模块名.xxx=>$store.state.setting.desc也可以通过 mapState 映射

十六、Vuex 模块化的使用小结

1.直接使用

  1. state => $store.state.模块名.数据项名
  2. getters => $store.getters[‘模块名/属性名’]
  3. mutations => $store.commit(‘模块名/方法名’, 其他参数)
  4. actions => $store.dispatch(‘模块名/方法名’, 其他参数)
