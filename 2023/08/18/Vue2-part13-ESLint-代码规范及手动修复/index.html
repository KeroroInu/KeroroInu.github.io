Vue-part13-ESLint 代码规范及手动修复

代码规范：一套写代码的约定规则。例如：负值符号的左右是否需要空格？一句结束是否要加；

ESlint:是一个代码检查工具，用来检查你的代码是否符合指定的规则（你和你的团队可以自行约定一套规则）。在创建项目时，我们使用的是 JavaScript Standard Style 代码风格的规则

一、ESLint 代码规范

1.JavaScript Standard Style 规范说明

建议把：https://standardjs.com/rules-zhcn.html 看一遍，然后在写的时候，遇到错误就查询解决

下面是这份规则中的一小部分：

  • 字符串使用单引号 - 需要转义的地方除外

  • 无分号

  • 关键字后加空格 if(condition) {...}

  • 函数名后加空格 function name (arg) { ... }

  • 坚持使用全等 === 摒弃 == 一旦需要检查 null || undefined时可以使用 obj == null

  • ……

2.代码错误规范

如果你的代码不符合 standard 的要求，eslint 会跳出来提醒你

3.手动修正

根据错误提示来一项一项手动修正

如果你不认识命令行中的语法报错是什么意思，你可以根据错误代码（func-call-spacing，space-in-partents，……）去 ESLint 规则列表中查找其具体含义

打开 ESLint 规则表，使用页面搜索（Ctrl + F）这个代码，查找对改规则的一个释义

二、通过 eslint 插件来实现自动修正

Eslint 会自动高亮错误显示通用配置，eslint 会自动帮助我们修复错误

  • vscode 中安装
  • 如何配置
  • 注意：eslint 的配置文件必须在根目录，这个插件才能才能生效。打开项目必须以根目录打开，一次打开一个项目
  • 注意：使用了 eslint 校验之后，把 vscode 带的那些格式化工具全禁用了 Beatify
